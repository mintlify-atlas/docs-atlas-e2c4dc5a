## Overview

ScooterShare delivers a complete scooter-sharing ecosystem with features designed for customers, administrators, and maintenance engineers. Built on Flask and Raspberry Pi, the platform combines IoT hardware integration with powerful web-based management tools.

## Core Features

### 1. User Registration & Authentication

Secure, role-based user management with encrypted credentials.

**Key Capabilities:**
- Multi-role registration system supporting Customer, System Admin, and Engineer account types
- BCrypt password hashing for secure credential storage (app.py:96-97)
- Flask-Session server-side session management
- Role-based access control with automatic dashboard routing
- Persistent login state across requests

**Booking Workflow:**

1. **Book Scooter** (customer.py:90-154)

   - Awards 100 loyalty points

2. **Cancel Booking** (customer.py:198-221)
   - Updates booking status to "Cancelled"
   - Returns scooter status to "Available"
   - Makes scooter available for other users

```python
# Booking with automatic point rewards
@customer_page.route('/book_scooter/<int:scooter_id>/<int:user_id>', 
                     methods=['POST'])
def book_scooter(scooter_id, user_id):
    # Update scooter status
    cursor.execute(
        "UPDATE scooters SET Status = 'Occupying' WHERE ScooterID = %s",

    # Create booking
    cursor.execute(
        "INSERT INTO bookings (ScooterID, UserID, BookingTime,

        (scooter_id, user_id, 'Active')
    )
    
    # Award loyalty points
    points_to_increment = 100
```

### 4. IoT Lock/Unlock System

Real-time scooter control via Raspberry Pi integration.

**Key Capabilities:**
- Remote lock/unlock via web interface
- Balance verification before unlock
- Automatic charging on lock
- Location update on return
- Usage tracking on completion
- Separate controls for customers and engineers

**Customer Lock/Unlock Flow:**

<Steps>
  <Step title="Unlock Scooter">
    Customer requests unlock → System verifies sufficient balance → Lock status updated to "Unlocked"
  </Step>
  <Step title="Use Scooter">
    Customer uses scooter while booking remains active with unlocked status
  </Step>
  <Step title="Lock Scooter">
    Customer locks scooter → Balance charged → Booking marked "Completed" → Scooter status returns to "Available" → Usage record created
  </Step>
</Steps>

```python
# Unlock with balance verification (customer.py:573-604)
@customer_page.route("/customer_unlock_scooter/<int:scooter_id>/<int:booking_id>")
def customer_unlock_scooter(scooter_id, booking_id):
    # Check available balance
    cursor.execute(
        "SELECT AvailableMoney FROM users WHERE UserID = %s",
        (session.get('user_id'),)
    )
    available_money = cursor.fetchone()[0]
    
    cursor.execute(
        "SELECT CostPerTime FROM scooters WHERE ScooterID = %s",
        (scooter_id,)
    )
    cost_per_time = cursor.fetchone()[0]
    
    if available_money < cost_per_time:
        return "Not enough funds to unlock scooter. Please topup account"
    
    # Unlock scooter
    cursor.execute(
        "UPDATE bookings SET LockStatus = 'Unlocked' WHERE BookingID = %s",
        (booking_id,)
    )
```

<Warning>
  Customers must have sufficient balance to unlock a scooter. The system checks available funds before allowing unlock operations.
</Warning>

```python
@customer_page.route('/update_user_money', methods=['POST'])
def update_user_money():
```

### 7. Maintenance & Repair System

Comprehensive repair workflow with engineer notifications.

**Key Capabilities:**
- Scooter issue reporting by customers and admins
- Automatic status change to "To Be Repaired"
- Email notifications to all engineers via Mailgun
- Repair status tracking (In Progress/Complete)
- Repair history maintenance
- Engineer-specific dashboard for reported scooters

**Repair Workflow:**

<Steps>
  <Step title="Report Issue">
    Admin or customer reports scooter problem with description
  </Step>
  <Step title="Automatic Notification">
    System emails all engineers with scooter details and issue description
  </Step>
  <Step title="Engineer Response">
    Engineer views reported scooters and updates repair status
  </Step>
  <Step title="Status Management">
    Engineer toggles status between "In Progress" and "Complete"
  </Step>
</Steps>

```python
# Report scooter issue (admin.py:416-478)
@admin_page.route('/add_repair/<int:page_id>', methods=['POST'])
```
