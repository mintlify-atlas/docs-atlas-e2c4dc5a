## GET /engineer_lock_unlock

Retrieves active bookings with both locked and unlocked scooters for engineer control interface.

### Database Queries

Makes internal requests to:
- `/engineer_active_bookings_unlocked` - Get unlocked active bookings
- `/engineer_active_bookings_locked` - Get locked active bookings

### Response

Returns rendered HTML template with lockable and unlockable scooter data.

## GET /engineer_unlock_scooter/\{scooter_id\}/\{booking_id\}

Unlocks a scooter for a specific booking. Engineers can remotely unlock scooters for maintenance or customer assistance.

### Path Parameters

<ParamField path="scooter_id" type="integer" required>
  The ID of the scooter to unlock
</ParamField>

<ParamField path="booking_id" type="integer" required>
  The ID of the active booking
</ParamField>

### Database Operation

**Query**: `UPDATE bookings SET LockStatus = 'Unlocked' WHERE BookingID = %s`

### Response

<ResponseField name="message" type="string">
  Success message confirming the unlock operation
</ResponseField>

### Response Example

```json
{
  "message": "Scooter Unlocked Successfully"
}
```

### Error Response

```json
{
  "error": "Error message description"
}
```

## GET /engineer_lock_scooter/\{scooter_id\}/\{booking_id\}

Locks a scooter and completes the booking. This endpoint performs multiple database operations to finalize a scooter rental.

### Path Parameters

<ParamField path="scooter_id" type="integer" required>
  The ID of the scooter to lock
</ParamField>

<ParamField path="booking_id" type="integer" required>
  The ID of the active booking to complete
</ParamField>

### Query Parameters

<ParamField query="newLocation" type="string">
  The new location where the scooter is being locked
</ParamField>

### Database Operations

1. **Update Lock Status**: `UPDATE bookings SET LockStatus = 'Locked' WHERE BookingID = %s`
2. **Complete Booking**: `UPDATE bookings SET BookingStatus = 'Completed' WHERE BookingID = %s`
3. **Set Scooter Available**: `UPDATE scooters SET Status = 'Available' WHERE ScooterID = %s`
4. **Update Location**: `UPDATE scooters SET Location = %s WHERE ScooterID = %s`
5. **Record Usage**: `INSERT INTO scooter_usage (ScooterID, UserID, TimeUsed, UsageDate, DistanceCovered) VALUES (%s, %s, %s, DATE(NOW()), %s)`

### Response

<ResponseField name="message" type="string">
  Success message confirming the lock operation
</ResponseField>

### Response Example

```json
{
  "message": "Scooter Locked Successfully"
}
```

### Error Response

```json
{
  "error": "Error message description"
}
```

### Notes

<Info>
This endpoint uses the session user_id to record usage data. The TimeUsed and DistanceCovered are currently hardcoded to 10 in the implementation.
</Info>

## GET /engineer_active_bookings_unlocked

Retrieves all active bookings where scooters are currently unlocked.

### Database Query

**Query**: `SELECT BookingID, ScooterID, BookingTime, BookingStatus FROM bookings WHERE BookingStatus = 'Active' AND LockStatus = 'Unlocked'`

### Response

<ResponseField name="booking_history" type="array">
  Array of active unlocked bookings
  
  <ResponseField name="BookingID" type="integer">
    Unique identifier for the booking
  </ResponseField>
  
  <ResponseField name="ScooterID" type="integer">
    ID of the scooter in this booking
  </ResponseField>
  
  <ResponseField name="BookingTime" type="string">
    Timestamp when the booking was created (format: "YYYY-MM-DD HH:MM:SS")
  </ResponseField>
  
  <ResponseField name="BookingStatus" type="string">
    Status of the booking (will be "Active")
  </ResponseField>
</ResponseField>

### Response Example

```json
{
  "booking_history": [
    {
      "BookingID": 501,
      "ScooterID": 3,
      "BookingTime": "2026-03-04 14:30:00",
      "BookingStatus": "Active"
    }
  ]
}
```

## GET /engineer_active_bookings_locked

Retrieves all active bookings where scooters are currently locked.

### Database Query

**Query**: `SELECT BookingID, ScooterID, BookingTime, BookingStatus FROM bookings WHERE BookingStatus = 'Active' AND LockStatus = 'Locked'`

### Response

<ResponseField name="booking_history" type="array">
  Array of active locked bookings
  
  <ResponseField name="BookingID" type="integer">
    Unique identifier for the booking
  </ResponseField>
  
  <ResponseField name="ScooterID" type="integer">
    ID of the scooter in this booking
  </ResponseField>
  
  <ResponseField name="BookingTime" type="string">
    Timestamp when the booking was created (format: "YYYY-MM-DD HH:MM:SS")
  </ResponseField>
  
  <ResponseField name="BookingStatus" type="string">
    Status of the booking (will be "Active")
  </ResponseField>
</ResponseField>

### Response Example

```json
{
  "booking_history": [
    {
      "BookingID": 502,
      "ScooterID": 7,
      "BookingTime": "2026-03-04 13:15:00",
      "BookingStatus": "Active"
    }
  ]
}
```

### Notes

<Warning>
There is a SQL syntax error in the original code - the query contains an extra "AND" at the beginning of the WHERE clause which may cause issues.
</Warning>
